    üîÑ Duplicates vs References: Understanding the Difference in Power BI

    One of the most misunderstood concepts! Duplicating vs Referencing queries has major implications once your data is in the model.

    What Happens in Power Query:

    ‚úì Reference Query
    • ‚úì Points to the original query
    • ‚úì Source query runs ONCE
    • ‚úì Adds transformations on top
    • ‚úì Efficient and fast
    • ‚úì Changes to source flow through
    ‚úó Duplicate Query
    • ‚úó Creates independent copy
    • ‚úó Source query runs TWICE
    • ‚úó Completely separate lineage
    • ‚úó Slower, more resource intensive
    • ‚úó Changes to source DON'T flow through

    What Happens in the Power BI Model:

    üìä Reference Behavior
    • ‚úì Creates separate table in model
    • ‚úì Each table takes memory independently
    • ‚úì Can create relationships between them
    • ‚úì Refresh is optimized (source loads once)
    ‚ö†Ô∏è Duplicate Behavior
    • ‚ö†Ô∏è Also creates separate table in model
    • ‚ö†Ô∏è Same memory footprint as reference
    • ‚ö†Ô∏è Can also have relationships
    • ‚ö†Ô∏è Slower refresh (source loads twice)
    üí° Key Insight: In the final model, both References and Duplicates create separate tables with separate memory. The BIG difference is refresh performance and maintainability during development!
    ‚ö†Ô∏è Best Practice: Always use Reference unless you specifically need an independent copy that won't change when the source changes. Your refresh times will thank you!
